# ADR-001: Формат денежных значений в API

| Метаданные | Значение |
|---|---|
| **Дата** | `2024-06-01` |
| **Статус** | `🟢 Принято`  |
| **Авторы** | `@Goluboy` |
| **Затронутые компоненты** | `OrderService, Kafka, Frontend, OpenAPI` |

---

## Контекст
Нужно передавать цены в API. Варианты:
- `number` (float/double) - просто, но потеря точности в JS
- `string` ("1299.99") - точно, но нужно парсить
- `integer` в копейках - точно, но неудобно для отображения

## Решение
Использовать объект `Money` с полем `amount` как **строка** с паттерном `^\d+\.\d{2}$`:

``` yaml
Money:
  type: object
  required: [amount, currency]
  properties:
    amount:
      type: string
      pattern: '^\d+\.\d{2}$'
      example: "1299.99"
      description: Сумма в формате строки для сохранения точности (2 знака после запятой)
    currency:
      type: string
      pattern: '^[A-Z]{3}$'
      default: RUB
      example: "RUB"
      description: Код валюты по стандарту ISO 4217 (например, "RUB", "USD", "EUR", "KZT")
```

## Последствия
### Позитивные
- Гарантирована точность между разными языками

### Негативные
- Нужно парсить строку в число на клиенте
- Полный запрет на обработку цен на клиенте

### Влияние на Frontend
- Добавить парсер

## Рассмотренные альтернативы
| Вариант | Почему отклонён |
|---|---|
| `int в копейках` | `Нужна логика перевода обратно в валюту, отдельная обработка операций, чтобы не потерять даныне` |
| `number + format: double` | `Риск потери данных при переводе` |
