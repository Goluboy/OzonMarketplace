## Контекст
Dapper - Micro ORM, не реализующая механизм ChangeTracker и паттерн UnitOfWork в рамках одного HTTP запроса.
Было принято решение реализовать данный паттерн для реализации атомарности обновления нескольких таблиц в одной транзакции.

## Решение
Единый UnitOfWork Scoped сервис, реализующий два различных интерфейса для изоляции использования. Для соблюдения чистоты архитектуры 
были реализованы интерфейсы:
- `IUnitOfWork`, позволяющий работать с различными транзакциями в Application слое
- `IDbSession`, позволяющий работать с транзакцией в репозиториях в Infrastructure слое

## Последствия
### Позитивные
- Соблюдена чистота архитектуры
- Реализован механизм атомарного обновления нескольких таблиц

### Негативные
- Накладные расходы

## Рассмотренные альтернативы
| Вариант | Почему отклонён |
|---|---|
| `EF Core` | `Тяжелый ORM, много накладных расходов, не подходит по стеку` |
