## Контекст
В классическом сценарии использовани сервиса продуктов бОльшая часть запросов к сервису будет на чтение (порядка 95%), остальная часть запросов на запись(5%).
В связи с этим было принято решение внедрить CQRS паттерн в Infrastructure и Application слои сервиса Product для разделения Flow запросов на чтение и запись.

## Решение
Разделить обработку запросов на чтение и запись в Application сервисах и Infrastructure репозиториях. Выделить разные модели для обработки запросов.
Репозитории на чтение должны быть максимально эффективными, не должны включать в себя использование паттерна UnitOfWork.
