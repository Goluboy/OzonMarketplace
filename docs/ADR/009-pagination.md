## Контекст
Для обеспечения быстродействия системы и оптимального выполнения запросов нужно выбрать способ пагинации в Product сервисе для получения карточек продукта.

## Решение
Использовать курсорную пагинацию, кодируя курсор в base64 формат при передаче его клиенту.
Курсор как таковой использует формат sortValue|id, где id - индентификатор последней позиции, sortValue - значение из сортируемого столбца для последней позиции.
Кодировка используется для инкапсуляции логики пагинации, что повышает безопасность системы.

## Последствия
### Позитивные
- Более высокая производительность по сравнению с offset пагинацией
- Инкапсуляция логики пагинации

### Негативные
- Более сложная реализация

### Влияние на Frontend
- Упрощение реализации пагинации

## Рассмотренные альтернативы
| Вариант | Почему отклонён |
|---|---|
| `offset` | `Менее производительный вариант пагинации, поскольку БД нужно будет считать общее количество элементов для подсчёта и отсеивать ненужное количество` |
