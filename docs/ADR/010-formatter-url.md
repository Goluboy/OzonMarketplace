# ADR-010: Форматирование URL в относительный формат

| **Затронутые компоненты** | `S3, ProductService` |

---

## Контекст
Хранение абсолютных путей в БД мешает горизонтальному масштабированию сервиса.

## Решение
Было принято решение реализовать сервис по форматированию URL из абсолютного в относительный и обратно.
Универсальные методы по преобразованию лежат в общей S3 DLL, что позволяет унифицировать это решение. 
Для декомпозиции бизнес-сервисов было принято решение добавить дополнительный сервис-обёртку в Applicaiton слой для форматирования URL.

## Последствия
### Позитивные
- Сервис по прежнему Stateless
- Поддержка горизонтального масштабирования

### Негативные
- Больше аллокаций, следовательно ниже производительность
- Загрязнение слоя Application и бизнес-сервисов

## Рассмотренные альтернативы
| Вариант | Почему отклонён |
|---|---|
| `Хранение абсолютный путей` | `Не позволяет горизонтально масштабировать сервис, привязывается к определенному S3 хранилищу` |
| `ActionFilter` | `Хорошее решение, но не явное. Можно легко забыть про него в случае замены` |
| `JsonSerializer` | `Специфичная настройка сериализатора ещё менее явное решение, перекладывать часть сценария обработки на него - антипаттерн` |
