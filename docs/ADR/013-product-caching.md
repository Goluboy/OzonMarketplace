## Контекст
Для оптимизации работы каталога важно применять производительные решения при получении карточек товара и полного описания товара. 
Нужно учесть динамический механизм фильтрации, что подразумевает фильтрацию на стороне БД. 

## Решение
Для оптимизации передачи данных по сети было принято решение получать из БД список Id нужных товаров после фильтрации. Каждый запрос, требующий фильтрации
будет проходить через БД. В Redis храняться карточки товаров храняться отдельно от детального описание. По Id сервис подгружает закешированные карточки, после чего
на этапе сравнения формируется список Id отсутсвующих карточек (Miss), далее единым запросом в БД мы получаем актуальные карточки товаров, после чего кешируем их и отдаём пользователю.
На уровне Application сервиса происходит сортировка готового списка товаров за O(N) на основе словаря.

## Последствия
### Позитивные
- Снижение нагрузки на БД
- Уменьшение времени ответа сервера

### Негативные
- Повышение сложности кода
