## Контекст

Проблемы текущей Portainer инфраструктуры:
1. Portainer без root-прав, нужно идти на компромиссы: 
    1. нет локальных конфигов
    2. отказ от Promtail 
    3. отказ от bind-mounts
2. Гонка контейнеров из-за отсутствия depends_on
3. Использовать можно только готовые образы, если нужны конфиги
4. Нет полного контроля: ssh, /var/log. Только через UI Portainer 

## Решение

Переход на облачный хостинг, арендовать сервер с root доступом и развернуть docker-compose с минимальными изменениями

## Последствия
### Позитивные
- Быстрый старт и низкий порог входа
- Полный доступ
- БЕСПЛАТНО (есть 4000 для новых аккаунтов)
- Работает из коробки, потому что запускает обычный docker-compose с root 
- большая инфраструктура для расширения

### Негативные
- ПЛАТНО (после 4000)
- необходимо вручную поддерживать при падении

### Влияние на Frontend
- никакого

## Рассмотренные альтернативы
| Вариант | Почему отклонён |
|---|---|
| `Yandex Cloud + Managed services` | `Избыточно, долгая настройка` |
| `Kubernetes` | `оверкил для маленького преокта` |

в целом остальные варианты добавляют абстракции/функции, которые не пригодятся. Для текущего прототипа проще развернуться напрямую + есть бесплатный тариф.
